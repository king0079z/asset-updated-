RFID Infrastructure Architecture

Huawei AP + RFID Reader Chip Integration

The RFID infrastructure is built on a co-located architecture where each Huawei Wi-Fi 6 Access Point hosts a dedicated RFID Reader Chip Module (RFID-AP integration unit). This approach eliminates the need for separate RFID infrastructure cabling, reducing installation cost and complexity while leveraging the existing PoE network distribution.

🔌

RFID-over-AP Architecture: How It Works

Each Huawei AP has an RFID reader chip/module physically mounted to or integrated with it. The RFID module receives tag signals from nearby active tags and passes the raw RFID events to the AP over an internal data bus or dedicated Ethernet port. The AP uplinks all data — both Wi-Fi traffic and RFID events — over the existing PoE LAN switch to the RFID Middleware Bridge, which normalizes the LLRP protocol messages and forwards them as JSON events to the AssetXAI Platform API via HTTPS.

Integration Architecture

D365 · Azure AD · M365 Email
Microsoft Dynamics 365 Integration

AssetXAI integrates with D365 (Finance & Operations or Business Central) via the standard OData v4 REST API. The integration is bidirectional — assets created or modified in AssetXAI are pushed to D365 Fixed Assets module; depreciation schedules, asset values, and procurement orders flow back from D365 to AssetXAI for enriched reporting.

Data EntityDirectionFrequency
Asset Register (create/update/disposal)→ D365Real-time
Fixed Asset Depreciation ValuesD365 →Nightly batch
Asset Location & Status Updates→ D365Real-time
Purchase Orders (procurement)D365 →On-demand
Disposal Records & Write-offs→ D365Real-time
User / Department AssignmentsD365 →Daily sync
Azure Active Directory SSO

Users authenticate via Azure AD using the OpenID Connect (OIDC) protocol. On first login, AssetXAI auto-provisions the user with default role permissions. MFA enforcement is delegated to Azure AD Conditional Access policies, keeping all authentication controls within QRCS's existing security governance.

🔐

SSO Flow

  • User clicks "Sign in with Microsoft" on AssetXAI login page
  • Redirected to QRCS Azure AD tenant for authentication + MFA
  • Azure AD issues ID token (JWT) to AssetXAI callback endpoint
  • AssetXAI validates token, maps AD groups to AssetXAI roles
  • User is logged in — session tied to Azure AD lifecycle (auto-expiry on deprovisioning)
📨

Microsoft 365 Email-to-Ticket

  • Dedicated shared mailbox (e.g., support@qrcs.org.qa) monitored by AssetXAI
  • Inbound emails auto-create tickets with subject as title, body as description
  • Reply-to-ticket threading: staff replies within ticket update the email thread
  • Outlook Add-in: raise & track tickets directly from inbox pane
