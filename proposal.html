How the System Works — Complete Operational Overview

End-to-End User Journey

AssetXAI is a fully integrated enterprise platform that connects physical assets (tagged with Active or Passive RFID chips), the RFID reader infrastructure (Huawei AP + RFID reader module), field technicians (handheld devices), management (web dashboard), and backend systems (D365, Azure AD) into one seamless operational workflow. Here is how the entire system operates from asset registration through to reporting.

Step 1 — Asset Registration & RFID Tagging
📦

New Asset Onboarded

When a new asset arrives (e.g., laptop, medical equipment, furniture), a staff member or technician registers it in AssetXAI via:

  • Web portal — fill asset form (name, type, serial, model, department, location, purchase date, value)
  • Mobile/handheld app — scan existing barcode/QR from asset and auto-populate fields
  • Bulk Excel import — for large asset register migrations (thousands of assets at once)
🏷

RFID Tag Attached & Bound

After registration, a physical RFID tag is attached to the asset:

  • Active RFID tag (433 MHz) for high-value or frequently moved assets — broadcasts ID continuously every few seconds, detected automatically in real time without scanning
  • Passive RFID tag (UHF 860–960 MHz) for furniture and lower-value items — read when scanned by handheld or passed through passageway reader
  • The tag EPC/ID is bound to the asset record in the database. Duplicate binding is rejected.
  • A QR/barcode label is also printed and affixed as a backup identification method

Step 3 — Automated Alerts & Event Handling
⚠️

Zone Boundary Alert

When an asset with an Active RFID tag is detected by a passageway reader leaving an authorized zone, AssetXAI instantly:

  • Records the exit event with timestamp and user present
  • Checks against zone access rules
  • Fires an alert notification to the manager
  • Optionally auto-creates a ticket for investigation
🔋

Low Battery / Tamper Alarm

When an Active RFID tag broadcasts a low-battery flag or a tamper sensor is triggered:

  • Tag event arrives at AssetXAI webhook
  • Alert rule matches → notification sent to maintenance team
  • Maintenance ticket auto-created with asset ID and location
  • Tag health dashboard updated
🔔

SLA Breach & Overdue Alerts

The built-in SLA Engine continuously monitors:

  • Open tickets approaching their SLA deadline
  • Borrowed assets past their return date
  • Warranty/service contract expiry dates
  • Scheduled preventive maintenance windows
  • Escalation at each breach threshold

Step 5 — Inventory Audit & Stocktaking

When management needs to conduct a periodic inventory audit, a technician uses the handheld device to walk through each room/zone. The process is fully guided by the Inventory Audit Center:

  • Pre-loaded scan list: System shows expected assets per zone based on last known locations
  • Handheld scanning: Technician scans passive UHF tags one by one or in bulk sweep; Active RFID tags are auto-detected by the fixed reader infrastructure
  • Real-time reconciliation: Each scan is checked against expected — Present / Missing / Unexpected alerts shown instantly
  • Offline mode: All scans queue locally if Wi-Fi drops. Synced automatically on reconnection
  • Report generation: Full audit report generated with discrepancies, timestamps, staff name, and asset details — exportable as PDF or Excel
  • Report completion: Management reviews and marks report "Complete". Report moves to Completed Reports archive

The Inventory Audit Center dashboard gives management a real-time overview:

  • Total reports submitted vs. completed
  • Current open reports with status (In Review, Tickets Raised, Completed)
  • Any tickets linked to audit findings
  • Asset discrepancy summary across all 7 sites
  • Handheld worker activity and scan counts per session
🖨

Print-Quality Audit Report

One-click print report generation includes all asset details, scan results, discrepancies, linked tickets, timestamps, submitting staff member, and management sign-off fields — fully formatted for external auditors.

Step 6 — ERP & Microsoft 365 Ecosystem Integration
🟦

Dynamics 365 — Bidirectional Sync

  • Asset created in AssetXAI → automatically pushed to D365 Fixed Assets module via OData REST API
  • Asset disposed/transferred in AssetXAI → D365 status updated in real time
  • Depreciation schedules and values fetched from D365 nightly → displayed in AssetXAI financial reports
  • Purchase orders from D365 procurement → pulled into AssetXAI for asset creation workflow
  • Sync errors logged and surfaced on admin dashboard with retry option
🔐

Azure AD SSO — User Authentication Flow

  • User opens AssetXAI and clicks "Sign in with Microsoft"
  • Redirected to QRCS Azure AD tenant for credentials + MFA
  • Azure AD issues signed JWT token → returned to AssetXAI callback
  • Token verified → user auto-provisioned on first login with default role
  • Azure AD admin deactivates user → session expires automatically at next token refresh
  • Admin can also use Magic Login Link for password reset without SMTP dependency
Step 7 — AI Analytics & Predictive Intelligence
📈

Consumption Analysis

AI engine analyses historical asset usage, maintenance costs, and food/supply consumption. Generates monthly forecasts, identifies anomalies, predicts budget requirements for the coming quarter, and highlights over-purchasing by site.

🔧

Predictive Maintenance

ML model analyses asset health scores, historical failures, and usage patterns to predict when an asset is likely to fail. Proactively creates scheduled maintenance tasks before failures occur, reducing unplanned downtime by up to 40%.

🗓

AI Task Planning

AI engine automatically suggests maintenance tasks, inspection schedules, and preventive actions based on asset health scores, expiry dates, and seasonal patterns. Tasks auto-assigned to available technicians with estimated effort.

Deployment Options — How the System Runs On Your Infrastructure

On-Premise (Recommended for QRCS): The entire AssetXAI stack is packaged as Docker containers and deployed on QRCS-owned servers. This means:

  • No data ever leaves QRCS premises — full data sovereignty
  • Database runs locally: MongoDB (local) or PostgreSQL on QRCS server
  • RFID Middleware Bridge runs as a local Docker container on each site's server or a central hub
  • Web application served from QRCS internal network or DMZ-exposed for internet access
  • Authentication via Azure AD or local auth — no cloud dependency required
  • Email notifications via local Exchange SMTP server

Cloud (If Preferred): Identical application code deployed on Vercel Edge + Supabase PostgreSQL cloud infrastructure. Benefits:

  • Zero infrastructure management for QRCS IT
  • Automatic scaling during stocktaking events with all 920 users active
  • Managed database backups, HA, and DR by Supabase (SOC 2 certified)
  • Data stored in GCC/Qatar region by default
  • Global CDN — fast access from all 7 sites regardless of WAN bandwidth
  • Same codebase — switching between cloud and on-premise requires only config change

RFID Infrastructure Architecture

Huawei AP + RFID Reader Chip Integration

The RFID infrastructure is built on a co-located architecture where each Huawei Wi-Fi 6 Access Point hosts a dedicated RFID Reader Chip Module (RFID-AP integration unit). This approach eliminates the need for separate RFID infrastructure cabling, reducing installation cost and complexity while leveraging the existing PoE network distribution.

🔌

RFID-over-AP Architecture: How It Works

Each Huawei AP has an RFID reader chip/module physically mounted to or integrated with it. The RFID module receives tag signals from nearby active tags and passes the raw RFID events to the AP over an internal data bus or dedicated Ethernet port. The AP uplinks all data — both Wi-Fi traffic and RFID events — over the existing PoE LAN switch to the RFID Middleware Bridge, which normalizes the LLRP protocol messages and forwards them as JSON events to the AssetXAI Platform API via HTTPS.
